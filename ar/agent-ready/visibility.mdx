يتيح مكوّن `Visibility` لصفحة واحدة أن تخدم جمهورين: البشر الذين يقرؤون على الويب ووكلاء الذكاء الاصطناعي الذين يعالجون مخرجات Markdown. المحتوى الموسوم بـ `for="humans"` يظهر على الصفحة المعروضة ولكنه مستبعد من تصديرات Markdown. المحتوى الموسوم بـ `for="agents"` يكون مخفيًا على الويب ولكنه مضمّن في Markdown.

هذا يعني أنه يمكنك كتابة تعليمات حوارية ومرتكزة على واجهة المستخدم للبشر وتعليمات منظّمة قابلة للقراءة آليًا للذكاء الاصطناعي — دون الحاجة إلى الحفاظ على صفحتين منفصلتين.

## كيف يعمل

يقدّم Mintlify كل صفحة على عنواني URL:

- `https://docs.example.com/page` — الموقع المعروض، للبشر
- `https://docs.example.com/page.md` — Markdown الخام، لوكلاء الذكاء الاصطناعي

يتحكّم `Visibility` في ما يظهر على كل URL.

## مثال

```mdx
<Visibility for="humans">
  <Steps>
    <Step title="Open the dashboard">
      Navigate to [dashboard.example.com](https://dashboard.example.com) and sign in.
    </Step>
    <Step title="Create an API key">
      Go to **Settings → API Keys** and click **New key**.
    </Step>
  </Steps>
</Visibility>

<Visibility for="agents">
  To authenticate, send a POST request to `/v1/auth/token` with the user's
  email and password in the request body. The response includes a bearer token
  valid for 24 hours. Pass this token as `Authorization: Bearer <token>` on
  all subsequent requests.
</Visibility>
```

يرى القرّاء من البشر شرحًا تفصيليًا لواجهة المستخدم خطوة بخطوة. بينما يحصل وكلاء الذكاء الاصطناعي الذين يعالجون Markdown على تعليمات API المباشرة بدلاً من ذلك.

## الخصائص (Props)

<ResponseField name="for" type='"humans" | "agents"' required>
  - `"humans"` — مرئي على صفحة الويب المعروضة، مستبعد من تصدير `.md`
  - `"agents"` — مخفي على صفحة الويب المعروضة، مضمّن في تصدير `.md`
</ResponseField>

## الدمج مع Prompt

يكمّل `Visibility` و`Prompt` بعضهما البعض بشكل جيد. استخدم `Visibility` لمنح الوكلاء سياقًا منظّمًا، ثم استخدم `Prompt` لمنح القرّاء من البشر مطالبة يمكنهم استخدامها لسؤال الذكاء الاصطناعي حول المحتوى نفسه.

```mdx
<Visibility for="agents">
  ## Authentication schema
  - Method: Bearer token
  - Header: `Authorization: Bearer <token>`
  - Token TTL: 86400 seconds
  - Refresh endpoint: POST /v1/auth/refresh
</Visibility>

<Prompt description="Ask about authentication" icon="lock">
  Explain how authentication works in this API and show me example code for
  getting a token and making an authenticated request.
</Prompt>
```
